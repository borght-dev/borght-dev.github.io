# Koen's blog style guide

Reference document for blog-drafter, blog-polish, blog-interview, blog-editor.
This is HOW Koen writes, with examples from posts he's already shipped.

## Voice in one paragraph

Direct, dev-to-dev, no marketing layer. Writes like he's explaining something to a colleague over coffee, not presenting at a conference. Comfortable being specific about tools and frustrations. Comfortable saying "I was wrong". Dutch directness, but in English. Fragments and short sentences are part of the rhythm, not accidents.

## Sentence patterns Koen uses

- Opens with the situation, not a hook: "It all started when I started working on..."
- Drops into informal asides mid-paragraph: "in my experience", "(read: League of Legends)"
- Uses italics for emphasis sparingly: "I was going _fast_"
- Self-aware confessions: "I am not great at keeping track of things"
- Casual technical: "the noisy work (polling, parsing, git)" not "performance-critical operations"

## Sentence patterns Koen does NOT use

- "In today's fast-paced world..."
- "Let's dive into..."
- "It's not just X, it's Y"
- "The truth is..."
- Three-adjective stacks: "fast, reliable, and scalable"
- Rhetorical questions as section openers

## Paragraph rhythm

Mix of:
- 1-sentence paragraphs for emphasis
- 3-4 sentence paragraphs for explanation
- Bullet lists for trade-offs and choices
- Code blocks and inline `backticks` for anything technical

Avoid: 5+ sentence dense paragraphs without a break.

## Banned vocabulary (silent fix)

leveraging, robust, seamless, delve, dive into, dive deep, tapestry, landscape, navigate, journey, unlock, empower, supercharge, game-changer, crucial, essentially, fundamentally, ultimately, certainly, indeed, moreover, furthermore, nevertheless, additionally

## Banned phrases (silent fix)

"it's worth noting", "keep in mind", "that being said", "with that in mind", "in essence", "in today's fast-paced", "at the end of the day", "in conclusion", "to summarize", "as we explore", "in this post we'll"

## Punctuation

- No em dashes. Use comma, period, or parentheses.
- Oxford comma: yes
- Headings: sentence case, not Title Case
- Inline code in backticks for: file paths, commands, code identifiers, error messages, keyboard shortcuts

## Things to keep, not "fix"

- Fragments
- Sentence-ending prepositions
- Mid-sentence asides in parentheses
- Contractions (I'm, don't, can't)
- Strong opinions stated flatly
- Self-deprecation
- Technical jargon the audience knows
